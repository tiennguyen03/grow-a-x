# PastEpics — archived epic-doc versions

When an epic doc in [`../EPICS/`](../EPICS/) is **meaningfully revised**, the
previous version of the whole file is copied here *before* the edit, so earlier
states stay easy to find without digging through git history.

**Naming:** `Epic<N>.<YYYY-MM-DD>.md`
(append `-b`, `-c`, … if the same epic is archived more than once in a day).

Use this when the change spans the epic — title, status, the tier/overview
tables, or several sprints at once. For a change scoped to a single sprint
section, archive just that section to [`../PastSprints/`](../PastSprints/)
instead.

> This is a manual mirror for convenience — **git history is still the source of
> truth.** See "Documentation convention" in `CLAUDE.md`.
